# RWA.LAT 国际化与区域化规范

版本：1.0  
首发语言：英语 `en`、简体中文 `zh-CN`、印地语 `hi`、西班牙语 `es`、阿拉伯语 `ar`、法语 `fr`、葡萄牙语 `pt`。

## 1. 语言行为

首次访问顺序：已保存偏好 → 浏览器语言 → 英语。用户可在欢迎页和设置页切换，立即生效并持久化。切换语言不改变国家、产品资格、计价币或法律文件版本。

阿拉伯语启用 `dir=rtl`；数字、USDT 代码、钱包地址和交易哈希保持从左到右。品牌 RWA.LAT 不翻译。

## 2. 翻译范围

必须覆盖：导航、按钮、错误、空状态、Guest、认证、首页、投资、详情、订单、钱包、AI、持仓、通知、KYC、安全、推荐、客服、设置、营销授权、官方验证与反诈骗。

产品名称和发行主体默认不翻译；产品副标题、收益来源、费用和风险可本地化，但必须对应同一内容版本。

## 3. 数字与时间

- 资产统一以 USDT 计价，同时显示 USD 参考值。
- 金额使用当前语言分隔符，资产代码不翻译。
- 百分比不得因语言转换改变数值。
- 时间存储 UTC，展示用户时区，并同时显示数据更新时间。
- 钱包地址、订单号和引用号不进行双向文本重排。

## 4. 文案质量

禁止仅机器翻译后直接上线。收益、风险、费用、赎回、AI、KYC、隐私和投诉文案需母语复核。缺失翻译时回退英语，同时记录缺失 key；生产环境不显示 key 名称。

## 5. 技术约定

- 语言 key 使用稳定语义名，不以英文句子作为 key。
- 所有页面通过统一 locale provider 获取语言。
- 路由不强制带语言前缀，偏好保存在本地并同步用户设置。
- 服务端消息返回稳定 code，客户端翻译；外部合作方原文可附在本地化解释下。
- 每个版本执行伪本地化、RTL、长文案、字体缺字和 320px 屏宽测试。

## 6. Demo 验收

七种语言均可切换并刷新后保留；主要导航和核心交易流程不能出现英文残留；阿拉伯语布局镜像但图表、地址和金额保持可读；营销授权、风险提示和退出规则必须有完整翻译。
