# 产品披露、费用与全生命周期规范

版本：1.0  
适用：AI Compute、RWA、Global Stocks、Prediction Markets 及未来产品。

## 1. 每个产品必须展示的事实

1. 产品名称、产品 ID、数据更新时间和 Demo/Live 状态；
2. 平台运营方、实际发行方、SPV/合同相对方、托管方、执行方和数据方；
3. 法律形态：代销、平台代理、直接发行、债权/收益参与、代币化凭证或纯数据导流；
4. 收益来源及现金流扣减顺序；
5. 预计收益的假设、区间、是否年化及“不保证”提示；
6. 认购/交易费、管理费、业绩分成、网络费、点差、税费和第三方费用；
7. 最低额、最高额、募集规模、期限、结算频率；
8. 到期、提前退出、赎回窗口、排队、延期、暂停和最坏情形；
9. 地区、客户类别、KYC 与适当性要求；
10. 文件、投诉、争议和实际责任方。

## 2. 各产品法律与收益基线

| 产品 | 建议法律形态 | 收益来源 | 主要退出机制 |
|---|---|---|---|
| AI Compute | 外部 SPV/供应商发行的算力收入参与或设备租赁合同；平台分销/技术服务 | 经验证 GPU 小时收入，扣除电力、机房、调度、维护、保险和储备 | 月度/季度退出队列；依流动性，不保证即时退出 |
| RWA | 发行人/SPV 的证券、票据、基金份额或合同权利；平台仅在获准范围内分销 | 租金、售电、利息、应收账款或资产出售所得，依现金流瀑布 | 到期偿还或定期赎回窗口；可延期、减记或暂停 |
| Global Stocks | 持牌券商执行或合规代币化证券发行方提供；平台不自营撮合 | 标的证券价格变化、股息和公司行动 | 支持市场时段卖出，受流动性、停牌和结算周期限制 |
| Prediction | Polymarket 数据/合作执行入口；平台不自建市场或做市 | 市场价格变化及最终事件结果 | 结算前可按市场流动性退出；最终依 resolution source |

## 3. 费用展示

费用必须在详情、订单确认和回执三处一致展示。顺序为：投入金额 → 平台费 → 第三方/网络费 → 实际投入 → 预计持有/管理费 → 退出费。百分比和绝对 USDT 同时显示，禁止使用“零费用”掩盖点差或合作方扣费。

## 4. 到期、提前退出和赎回状态机

`open → subscribed → active → exit_requested → queued → approved/rejected → settling → settled`。

必须支持：撤回截止时间、预计处理日、排队顺序、部分赎回、延期理由、暂停原因、费用、实际到账和争议入口。任何“预计 T+2”都不得表述为保证到账时间。

## 5. 订单确认

用户确认页必须固定展示产品、实际发行/执行方、金额、费用、报价有效期、地区/资格结果、退出规则和最大损失提示。AI 推荐订单还需展示建议版本及用户对参数的修改。订单使用幂等键，失败、部分成交和超时均有独立状态页和回执。
