# RWA.LAT H5 上线验收清单

版本：1.0。每项记录通过时间、构建版本、验证人和证据链接。

## 1. 关键路径

- [ ] Guest 可进入 Home、Invest、详情、公开 AI Brief；受保护动作要求登录。
- [ ] 注册、登录、验证邮箱、恢复和钱包签名入口均有成功/失败/重试。
- [ ] 七种语言切换、持久化、RTL 和回退通过。
- [ ] 产品显示发行方、法律形态、收益来源、费用、期限、退出和风险。
- [ ] AI 建议显示免责声明，并记录版本与用户确认。
- [ ] 订单显示报价有效期、费用、状态、失败和收据。
- [ ] 钱包限 TRON、Ethereum、Arbitrum USDT；错误网络警告清晰。
- [ ] 推荐显示单层关系、奖励状态、规则和反作弊说明。
- [ ] 营销授权可独立开关并退订；服务通知不受影响。
- [ ] 客服可创建、查看和申诉工单；订单争议可附引用。
- [ ] 官方验证页与诈骗举报入口可达。

## 2. 页面与交互

- [ ] 所有固定 CTA 不遮挡内容、键盘、Safe Area 或底部导航。
- [ ] 320、375、390、430px 宽度无横向滚动。
- [ ] iOS PWA、Android Chrome 和桌面窄屏可用。
- [ ] 骨架、加载、离线、空状态和错误页存在。
- [ ] 视频有静态封面、静音自动播放策略和降级图。
- [ ] `prefers-reduced-motion` 下减少动画。
- [ ] 可点击区域至少 44×44px；键盘焦点可见。

## 3. 数据与安全

- [ ] Demo 和 Live 数据清晰区分。
- [ ] 不在前端包含真实密钥、助记词、私钥或后台 API Key。
- [ ] 价格、收益、概率和文件显示来源与更新时间。
- [ ] 所有写操作具备幂等键、鉴权和服务端资格校验。
- [ ] 钱包地址、订单、收益和推荐记录可审计。
- [ ] CSP、HSTS、安全 Header、依赖扫描和敏感信息扫描通过。

## 4. 构建与运行

- [ ] 根 H5 独立 `build` 通过。
- [ ] API `build` 和单元测试通过。
- [ ] Admin API `build` 和测试通过。
- [ ] Admin Frontend 独立类型检查和构建通过。
- [ ] PostgreSQL 迁移可在空库执行并完整回滚。
- [ ] 本地 Demo 验证脚本在服务器启动后通过全部路由。

## 5. 当前已知基线（2026-07-13）

- 根目录类型检查会扫描子项目，导致 Admin Frontend 路径别名和 API Jest 类型错误混入 H5；应拆分项目级检查。
- 根 H5 构建曾被另一个正在运行的 Next build 锁占，需在写入结束后重跑。
- `verify:demo` 依赖本地服务运行；未启动服务时全部 fetch 失败不代表页面本身失败。
- Admin Frontend 是独立 Next 14/React 18 项目，不应由根 Next 16/React 19 的 tsconfig 直接编译。
