# RWA.LAT 官方渠道与反诈骗规范

版本：1.0 Demo

## 1. 官方原则

RWA.LAT 工作人员绝不会通过私聊索取助记词、私钥、验证码、屏幕共享、远程控制或要求向个人地址转账。所有充值地址只在登录后的官方产品内展示。

## 2. 官方验证页内容

正式页面 `/security/official-channels` 应列出：

- 主域名与状态页；
- App Store / Google Play 官方发布者；
- X、Telegram、Discord、YouTube、LinkedIn 和微信公众号；
- 官方客服与安全邮箱；
- 已知假冒账号和钓鱼域名；
- 域名、邮箱或社交账号的复制验证功能。

当前所有账号使用 `[待填写]`，不得在生产环境展示虚构链接。

## 3. 举报流程

用户可提交平台、账号、URL、钱包地址、聊天截图、发生时间和损失情况。系统生成 `SCAM-YYYYMM-XXXX` 编号。P0/P1 举报进入安全团队并提示用户暂停转账、撤销可疑授权、修改凭证和联系相关托管/执法机构。

## 4. 社群管理

- 官方管理员使用统一头像、可验证用户名和角色标识。
- 关闭“管理员主动私聊投资”行为。
- 每日置顶反诈骗提示和唯一官网链接。
- 发现仿冒后保存证据、举报平台、加入阻断名单并通知受影响社群。
- 不公开用户 KYC、余额、地址归属或安全调查细节。

## 5. 产品内警告

新增地址、提现、授权和外链跳转时显示上下文警告。首次提现显示“官方不会要求你向指定地址转账”。高风险地址或新设备触发延迟和人工复核。
