# Guest、国家准入与产品限制政策

版本：1.0 Demo Policy  
原则：全球可浏览不等于全球可投资；任何产品的可执行状态必须由服务端实时判定。

## 1. Guest 模式

Guest 可访问：主页公开市场摘要、Invest 目录、产品详情、公开数据来源、教育内容、费用示例、风险披露、官方渠道与反诈骗页。

Guest 不可访问：个人余额、持仓、收益、交易记录、个性化 AI 建议、充值地址、提现、转账、认购、赎回、推荐奖励和工单详情。受限动作统一进入登录页，并保留原目标页用于登录后返回。

## 2. 三层准入模型

1. **地区层**：居住国、国籍、自报位置、IP 风险、制裁与受限地区；
2. **用户层**：年龄、KYC/KYB、投资者类别、制裁/PEP、风险承受、资金来源；
3. **产品层**：发行文件允许地区、客户类型、最低额、上限、销售限制、市场与合作方状态。

这是强制的**产品级地区限制**：同一用户可能可使用钱包，但只能查看美股或预测市场；不得用账户级“已通过”覆盖单个产品的限制。

统一结果：

| 状态 | H5 行为 |
|---|---|
| `available` | 显示执行按钮、费用和报价有效期 |
| `view_only` | 可看数据和文件，不显示真实执行 |
| `review_required` | 说明缺少资料并进入补充审核 |
| `restricted` | 说明不可用，不泄露制裁筛查细节 |
| `suspended` | 产品或合作方暂停，展示已有持仓处置路径 |

## 3. 初始地区策略

首版建议采用保守基线：

- 全球 Guest 浏览；
- 真实交易仅对白名单国家和完成 KYC 的用户开放；
- 美国、受制裁地区及合作方明确禁止的地区默认 `view_only/restricted`，除非取得专项法律意见；
- 中国大陆用户可访问中文公开内容，但真实产品可用性必须根据跨境、证券、虚拟资产和营销法律意见单独决定；
- Polymarket、美股、代币化证券和 RWA 各自维护独立地区矩阵，不能复用钱包地区结论。

以上是产品控制基线，不是最终法律结论。本地 Demo 在没有正式矩阵时返回带理由码的模拟判定，以完整展示可用、只读、补充审核和受限状态；生产资金与外部执行通道才采用“未获批准不执行”，不得影响公开浏览和离线演示。

## 4. 判定与证据

每次详情页打开和订单提交均记录：政策版本、用户/地区输入、产品 ID、判定结果、理由码、数据来源、时间和最终动作。后台政策变更需双人批准，支持预览、定时生效和一键回滚。

## 5. 异常处理

- VPN/IP 与 KYC 国家不一致：进入人工复核，不自动拒绝；
- 合作方临时下线：禁止新订单，持仓页显示结算和客服路径；
- 用户旅行：不自动改变法定居住地，但执行时重新检查当前位置限制；
- 规则不可用：Demo 返回模拟状态并继续展示流程；真实订单不发送至资金或外部执行通道，公开浏览保持可用；
- 申诉：生成工单并保留原判定版本，人工决定不得覆盖历史记录。
