## 六、注意事项

- **不要用 `wsl -- sudo service postgresql start`** — WSL systemd 不可靠。用 `pg_ctlcluster 16 main start/restart`
- **PG 重启后必须 3 秒内启动 Core API**，否则 PG 会掉线
- **后台进程受限** — 这个 AI 环境无法常驻进程。你在本地 PowerShell 直接 `node dist/main.js` 前台跑最稳
- **不修改现有源文件** — 硬性约束。所有改动用 `patch` 工具或新增文件
- **图标只用 lucide-react** — 禁止自造 SVG 组合图标
