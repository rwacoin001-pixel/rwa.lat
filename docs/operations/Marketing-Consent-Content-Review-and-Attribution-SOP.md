# 营销授权、内容审核与渠道归因 SOP

版本：1.0 Demo  
负责人：Growth Lead  
复核角色：Compliance Reviewer、Product Owner

## 1. 营销授权中心

用户可独立控制：产品与研究邮件、市场推送、活动邀请和合作内容。安全、订单、钱包、结算、条款和工单消息属于服务通知，不因营销退订而停止。

授权记录字段：`user_id`、渠道、授权状态、文案版本、来源页面、国家、时间、IP/设备摘要、撤回时间和处理结果。默认关闭营销开关，不使用预勾选。

退订要求：页面立即反馈；营销发送名单在 24 小时内停止；继续保存最小化抑制记录，避免再次误发。

## 2. 内容发布流程

1. 作者创建草稿并指定目标地区、渠道和有效期。
2. 系统扫描收益承诺、缺失风险、未批准合作方、过期数字和错误链接。
3. 产品负责人核对功能和数据。
4. 合规复核产品形态、地区、声明和 CTA。
5. 发布人确认最终版本、UTM 和归档快照。
6. 重大错误可立即撤回，并记录影响渠道和用户通知。

高风险内容必须双人批准：APY/收益、产品开放、合作方/牌照、用户案例、推荐奖励、预测市场、美股和 RWA 发行。

## 3. 内容留档

每一条对外材料保存：内容 ID、完整快照、语言、渠道、地区、作者、审批人、批准时间、发布时间、URL、数据来源、素材授权、风险文案版本、撤回时间和原因。保存期限由正式法律意见确定，Demo 建议不少于 5 年。

社群管理员的业务沟通也属于留档范围。禁止使用个人账号私下承诺收益或接收用户资产。

## 4. UTM 与邀请码

标准参数：

- `utm_source`：wechat、x、youtube、telegram、discord、linkedin、partner；
- `utm_medium`：organic、community、video、newsletter、referral；
- `utm_campaign`：`YYYYMM_topic_region`；
- `utm_content`：素材/版本标识；
- `invite_code`：用户或合作伙伴邀请码。

首次触点、最近触点和注册邀请码分别保存，邀请码不得覆盖原始来源。敏感 URL 不附带邮箱、钱包地址或 KYC 状态。

## 5. 推荐活动审核

发布前定义地区、参与资格、有效行为、奖励类型、上限、锁定期、撤销和反作弊规则。禁止多级返佣、保证奖励价值、要求用户提交朋友联系方式或以投资金额无限计提。

## 6. 事故处理

发现误导内容、错误收益、假合作方、错误地区链接或诈骗冒充时：15 分钟内暂停传播；1 小时内确定影响范围；4 小时内发布更正或风险通知；24 小时内完成根因和控制改进记录。
