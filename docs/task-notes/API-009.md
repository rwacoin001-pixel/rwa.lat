# API-009 Polymarket adapter increment

**Status:** In progress  
**Updated:** 2026-07-15

## 2026-07-15 Core API increment

- Added a typed Core API adapter for official Gamma keyset pagination and public CLOB V2 order books.
- Added persisted read endpoints under `/v1/polymarket` and an admin-session + `catalog.manage` protected sync endpoint.
- Added market freshness metadata, sync leases, consecutive failure state and read-only degraded behavior.
- Added normalized immutable event ingestion/deduplication for a future server-side WebSocket worker.
- Added a production startup rejection for `POLYMARKET_TRADING_ENABLED=true`; no private key, L1/L2 credential or order submission method exists in the adapter.

## Implemented

- Public Gamma market discovery is served through `GET /api/polymarket/markets`.
- `GET /api/polymarket/market/:id` now combines Gamma single-market metadata with the public CLOB YES-token order book, midpoint, spread and 96-point price history.
- The endpoint now returns server-side integration readiness without returning API keys, builder code, or any signer secrets.
- Relayer configuration is kept in `.env.local`, which is ignored by Git. `.env.example` documents the required server-only variable names.
- The current integration is read-only: market discovery remains live; orders, cancellations, relayer submissions, deposits, withdrawals, and wallet signatures are not implemented or enabled.

## Verified constraints

- Gamma market discovery is public; it does not require a Relayer API key.
- CLOB read data can be added without credentials. Relayer keys are reserved for relayed transaction flows and must remain server-side.
- Before any real trading can be enabled, the product still needs user wallet signing, regional eligibility, partner approval, order idempotency, audit logging, and explicit user confirmation.
