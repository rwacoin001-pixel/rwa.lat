# FE-010 订单确认前端备注

状态：部分完成（2026-07-12）

本轮已完成可运行的订单 Demo 状态：

- 报价以 45 秒倒计时展示；过期后必须刷新，且不会锁定资金。
- 金额、可用余额和费用会阻止不可用的提交。
- 风险披露为显式确认项；未确认时不允许继续。
- 提交时出现“记录 Demo 请求”的处理中状态，防止重复点击；随后展示带参考号、执行状态和下一生产门槛的回执。
- 所有资产类别，包括 Polymarket，均明确为只读/演示：不创建仓位、不修改余额、不发送外部订单；回执可跳转到支持入口。
- 新增 `/orders/processing`、`/orders/partial`、`/orders/failed`、`/orders/receipt`，覆盖检查时间线、部分状态比例、失败原因、资金未锁定说明、收据明细及支持入口。

未完成：

- API-008 的幂等订单、资金锁定、真实处理/部分成交/失败/取消、收据和争议状态机。
- API-009 / PARTNER-002 的 Polymarket 外部订单映射、状态回传与对账。

验证：构建通过后，从任意资产详情进入 `/orders/review` 即可验证报价、确认与回执交互。
