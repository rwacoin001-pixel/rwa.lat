# API-006 任务备注

## 依赖与范围
- 依赖：DB-004（资产/产品/报价模型）
- 范围：资产目录、披露文件、价格和数据新鲜度
- 核心约束（Task-Board）：四类资产统一读取模型；**数据过期不能下单**。

## DESIGN.md 阅读记录
- 已阅读 DESIGN.md 与 Task-Board 完成定义。API-006 的强约束为"数据过期不能下单"——在 `CatalogService.requireOrderableQuote()` 实现：产品须 `published` 且最新报价 `valid_until > now()`，否则抛 `QUOTE_STALE`(409) / `PRODUCT_NOT_PUBLISHED`(409)。

## 实现内容
- `catalog.service.ts`：
  - `listAssetClasses()` / `listProducts(assetClass?, state?)` / `getProduct(id)` / `listDisclosures(productId)`：四类资产统一读取。
  - `getLatestQuote(productId)`：返回报价 + `stale` 新鲜度标记。
  - `requireOrderableQuote(productId, now)`：下单前置校验，过期/未发布拒绝（供 API-008 订单复用）。
  - `recordSnapshot(quote)`：写入价格快照（审计/对账前置）。
- `catalog.controller.ts`：
  - `GET /v1/catalog/asset-classes`
  - `GET /v1/catalog/products?assetClass=&state=`
  - `GET /v1/catalog/products/:id`
  - `GET /v1/catalog/products/:id/disclosures`
  - `GET /v1/catalog/products/:id/quote`
- `catalog.dto.ts`：query 校验（assetClass 命名、state 枚举）。
- 错误码本地定义于 `catalog.errors.ts`（不入侵 `packages/contracts`）。
