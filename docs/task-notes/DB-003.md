# DB-003 钱包与双式账本模型实施备注

**状态：** 进行中  
**日期：** 2026-07-12  
**设计前置：** 已在开工前完整阅读 `DESIGN.md`。本任务仅建立数据库、共享状态词典和实库测试，不新增用户界面或概念图。

## 本轮完成

- 新增 `CreateWalletLedgerModel1783753800000` 迁移，覆盖托管钱包、钱包地址、链交易、充值、提现、内部转账、账本账户、记账凭证、分录、当前余额投影与余额快照共 11 张表。
- 金额统一使用 `numeric(78, 0)` 保存资产最小单位整数，并独立保存 `asset_decimals`；数据库约束拒绝零值、负值和浮点记账形态。
- 最低账户分类覆盖用户可用、锁定、待处理、平台清算、费用收入、应付收益、资产持仓成本和外部托管对账差异。
- 记账凭证使用唯一幂等键，并保留请求 ID、业务对象引用、操作者、原因码、冲正关系和审计元数据。
- 延迟约束触发器在事务提交边界验证每张凭证至少两条分录、只有一种资产/精度且借贷相等。
- 分录写入时锁定账户余额投影；用户账户禁止负余额，从数据库边界防止并发资金操作透支。
- 每条分录同步产生不可变余额快照；记账凭证、分录和快照均拒绝更新、删除与截断，修正只能新增冲正或调整凭证。
- 充值按 `network + transaction hash + output index + asset` 去重；提现和内部转账按用户与幂等键去重。
- `packages/contracts` 新增钱包、地址、链交易、充值、提现和内部转账的共享状态词典；`CONTEXT.md` 补充钱包与账本领域语言。

## 新增实库测试

`test/wallet-ledger-schema.integration.spec.ts` 覆盖：

1. 最小单位精确入账、当前余额与不可变快照。
2. 小数形式的最小单位金额被拒绝而不是静默舍入。
3. 非平衡记账凭证在提交时被数据库拒绝。
4. 用户可用余额不能被扣成负数。
5. 链上充值与提现请求的幂等去重。
