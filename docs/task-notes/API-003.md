# API-003 会话、二次验证、Passkey、设备与安全事件进度备注

**状态：** 进行中  
**负责人：** Codex  
**开始日期：** 2026-07-12  
**设计前置：** 已在开工前完整阅读 `DESIGN.md`。本任务没有新增视觉页面；它为个人中心的账户安全、设备管理、提现/白名单等敏感流程提供服务端安全边界。

## 已实现内容

- 增加 Bearer 会话守卫：会话必须处于 active 状态且未过期；过期会话被显式标记为 `expired`，并刷新 `last_seen_at`。
- 增加安全 API：会话列表、单个会话撤销、撤销其他会话、设备列表、设备撤销和当前设备信任。
- 新增 TOTP 认证器模型和 API：加密保存 Base32 密钥，首次验证码确认才激活；提供一次性恢复码，数据库仅保存 HMAC。
- 新增 Passkey/WebAuthn：真实使用 `@simplewebauthn/server` 生成/验证注册和断言选项；挑战在数据库加密保存，五分钟到期且只能消费一次；凭据公钥和签名计数器持久化。
- 新增五分钟敏感操作复验凭证：绑定用户和当前会话，可由 TOTP、恢复码或 Passkey 得到；当前设备信任已要求该凭证，后续钱包、白名单和订单确认必须复用该校验。
- 复用 DB-002 的不可变 `app.audit_logs` 记录安全操作，前端可通过安全事件接口读取最近 50 条记录。
- 在共享契约中登记安全错误码，并补充 Passkey 运行配置、API README 和单元测试。
