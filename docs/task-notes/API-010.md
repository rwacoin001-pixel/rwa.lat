# API-010 任务备注

## 依赖与范围
- 依赖（Task-Board 标注）：API-007、API-008。
- 范围（Task-Board）：持仓、收益、价格快照、历史表现和赎回。强约束："前端组合金额可追溯到不可变凭证"。
- 本轮务实切分：余额/持仓复用 `LedgerService.listUserBalances` 的**不可变账本投影**（不新建余额表，符合可追溯）；价格复用 `CatalogService` 的 `price_snapshots` 最新报价；仅新建 `position_snapshots`（历史表现溯源）+ `redemptions`（赎回请求状态机，执行链路留 API-008）。不依赖订单模块即可交付查询/快照/赎回请求能力。

## 本轮实现范围
- 新增独立 `apps/api/src/portfolio/` 模块（不修改任何现有代码；仅在 `app.module.ts` 注册一行 `PortfolioModule`）：
  - `portfolio.entities.ts`：`PositionSnapshot`（user_id/product_id/asset_code/quantity/unit_price/currency/valued_at/captured_at）、`Redemption`（完整赎回请求状态机：requested→queued→executing→completed / canceled / failed，含 order_id/request_id/各时间戳/reason_code）。
  - `portfolio.errors.ts`：`PortfolioError` 本地错误类（productNotFound / quoteStale / redemptionNotFound / redemptionStateInvalid）。
  - `portfolio.service.ts`：
    - `listPositions`：复用 ledger 不可变投影（available+locked），`source: 'immutable-ledger-projection'`。
    - `captureSnapshot`：落一个持仓时点快照，价格取自最新报价快照（stale 则拒）。
    - `history`：按 `captured_at` 聚合快照总价值，limit 1–100 钳制。
    - `requestRedemption`：创建赎回请求 + 预览估值，状态机初态 `requested`，**不执行资金操作**（AI/API 无资金权限，执行留 API-008）。
    - `listRedemptions` / `cancelRedemption`（仅 requested/queued 可取消 → canceled，置 canceled_at）。
  - `portfolio.dto.ts`、`portfolio.controller.ts`（`/v1/portfolio/*`：positions/snapshots/history/redemptions）、`portfolio.module.ts`（注册 LedgerModule + CatalogModule）。
- 数据库：`DB-010` 迁移 `1783767000000-create-portfolio-model.ts` 新建 `app.position_snapshots` + `app.redemptions` 两表。
- 实体 `default` 仅在 DB 层生效，service 创建均显式设状态/时间戳（与 API-012 同一真 bug 模式规避）。
