# API-007 总账与对账服务实施备注

**状态：** 部分完成  
**日期：** 2026-07-12  
**设计前置：** 已在开工前完整阅读 `DESIGN.md`。本任务只新增账本、对账数据库与 API，不新增界面。

## 本轮完成

- 新增用户账本读取接口，余额来自精确余额投影，历史返回不可变记账凭证及其用户相关分录。
- 新增提现结算：只有处于 `broadcast/confirming` 且链交易已确认的提现才能结算；锁定余额和托管清算账户生成平衡凭证后才标记完成。
- 新增提现失败退款：锁定余额通过新的 `withdrawal_refund` 凭证返回可用余额，重复请求由账本幂等键阻止重复分录。
- 新增托管余额对账：记录账本期望余额、供应方观察余额和精确差异；零差异标记匹配，非零差异自动创建案件。
- 对账路径从不直接修改 `ledger_account_balances` 或插入调整分录。
- 新增 `reconciliation_runs`、`reconciliation_cases`、`ledger_adjustment_requests` 三张表。
- 账本调整请求要求申请人与审批人不同；已过账调整必须关联新的不可变记账凭证。
- 用户公开接口只有 `GET /v1/ledger/balances` 和 `GET /v1/ledger/transactions`；结算、退款、对账和调整未在 `API-013` RBAC 完成前暴露为用户可调用的管理 API。

## 自动化验证

- 账本投影读取和历史上限。
- 对账差异创建案件且不改余额。
- 链交易未确认时提现结算回滚。
- 重复退款不重复写分录。
- 实库测试定义差异算术约束和四眼审批约束。

全量结果：`npm run lint` 通过；`npm test -- --runInBand` 54 通过、18 跳过。

## 尚未完成

- 当前机器没有 PostgreSQL/Docker/`TEST_DATABASE_URL`，新迁移、提现结算并发和对账约束尚未执行实库验收。
- 正式托管方对账文件/API、日终调度、重试/死信、差异通知和财务导出待供应方及 `INFRA-002/004`。
- 账本调整的申请、审批、驳回、过账 API 必须等 `API-013` RBAC 与四眼审批身份模型完成后开放。
- 多资产、订单、持仓、收益和结算分录依赖 `DB-004/API-008/API-010`。
