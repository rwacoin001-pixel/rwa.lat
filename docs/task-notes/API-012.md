## 本轮实现范围
- 在 DB-005 已建表（tickets/invitations/subscriptions/preferences）之上，新增 `UserOpsService` + `UserOpsController`：
  - 工单：`POST /v1/user-ops/tickets`、`GET /v1/user-ops/tickets`、`PUT /v1/user-ops/tickets/:id/close`（close 幂等，置 closed_at）
  - 邀请：`POST /v1/user-ops/invitations`（生成 token + sha256(token_hash) 落库）、`GET .../invitations`、`PUT .../invitations/:id/revoke`（pending→revoked 置 revoked_at）
  - 订阅：`GET /v1/user-ops/subscriptions`、`PUT /v1/user-ops/subscriptions/:id/cancel`（active→canceled 置 canceled_at）
  - 偏好：`GET /v1/user-ops/preferences`（无记录返回默认）、`PUT /v1/user-ops/preferences`（upsert locale/channels/communication_consent）
- 状态机与 DB-005 的 CHECK 互斥约束一致（close/cancel/revoke 均通过 service 显式设状态 + 时间戳，因实体 `default` 仅在 DB 层生效，视图层需显式赋值——已修 `createInvitation` 漏设 `state:'pending'` 的真 bug）。
- 复用 NotificationModule（不新建 module），未入侵 `packages/contracts`（错误码仍用 NotificationError 本地类）。
