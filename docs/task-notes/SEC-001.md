# SEC-001 — Admin identity and session source

- Status: pending review
- Implementer: Codex
- Date: 2026-07-15
- Design review: `DESIGN.md` read before this task. This task changes authentication behavior only and does not alter the locked H5 visual system.

## Changes

- Replaced the fixed administrator password, mock JWT, public API key, browser-stored administrator ID, and browser-supplied approval actor with a server-side administrator session.
- Added hashed, revocable, expiring `app.admin_sessions`; passwords use salted scrypt hashes, lock after configurable failed attempts, and record last login.
- Added Admin API `login`, `logout`, `refresh`, and `me` routes. Session tokens are stored only in the Admin Frontend BFF's HttpOnly, SameSite cookie; the browser uses same-origin `/api/admin/*` forwarding.
- Added an encrypted TOTP factor path. A pending, missing, invalid, or misconfigured MFA factor fails closed and never issues a session. MFA secrets are AES-256-GCM ciphertext and require the server-only `ADMIN_MFA_ENCRYPTION_KEY` when enabled.
- Core RBAC, data-governance, and object-storage administration now derive the actor from `AdminSessionGuard`. No request body or browser header chooses the administrator or approver.
- Added immutable audit entries for administrator login success/failure, MFA requirement, refresh, logout, and approval request/decision. Approval audit metadata includes server request ID, IP, and User-Agent.

## 2026-07-15 route-ownership hardening

- Portfolio, notification, ticket, invitation, reward, subscription and preference routes now derive the end-user subject from `SessionAuthGuard`; callers cannot select another user through a query parameter or request body.
- KYC decisions, risk-flag mutation, alert administration, notification creation, queue administration, approval/audit administration, object storage and data governance require an administrator session plus an explicit least-privilege permission.
- Missing RBAC permissions return HTTP 403 instead of being reported as a malformed request.
- Generic partner callbacks no longer accept caller-provided `signatureOk` or external IDs. The server verifies a timestamped HMAC-SHA256 signature, binds the partner, event type, event ID and payload, rejects requests outside a five-minute window, and fails closed when the partner secret is absent.
- Reading unprocessed callbacks is non-destructive. Callback processing state is no longer changed by an administrative list request.
- Demo administrator permissions are seeded only in non-production environments through the existing production-gated demo seed policy.
- Request ingress now applies bounded, per-IP fixed-window limits: 1,000 requests/minute globally, 5 authentication attempts/minute, 3 registrations/hour, 5 recovery requests/hour, 10 step-up attempts/minute and 300 generic partner callbacks/minute. Rejections return HTTP 429 with `Retry-After`.
- Prometheus metrics now export the application's actual private registry and require a constant-time compared bearer token in production. The obsolete unguarded duplicate metrics module and its extra dependency were removed.
- Production startup requires `METRICS_BEARER_TOKEN` and an explicit `TRUST_PROXY_HOPS` value. Node request/header/keep-alive timeouts, header count and requests-per-socket are bounded.
- Low-memory verification passed serially: TypeScript check, 24 Jest suites / 139 tests, targeted callback verification, the 1024MB-bounded production build and the production dependency audit (zero high/critical findings; one moderate finding remains).

## 2026-07-15 Admin API edge hardening

- Admin login is limited to five attempts/minute/IP and all Admin API traffic to 300 requests/minute/IP, in addition to the existing per-account failure lockout.
- Admin production startup requires a dedicated PostgreSQL production URL, exact HTTPS Admin CORS origins/API URL, explicit trusted-proxy hop count, `ADMIN_MFA_REQUIRED=true`, and a non-placeholder 32-byte base64 MFA encryption key.
- Production password-only login is no longer possible: an administrator without an enabled TOTP factor fails closed before session issuance.
- Admin request/header/keep-alive timeouts, header count and per-socket request count are bounded; login email/password input lengths are capped.
- Independent Admin API reads now have method-level RBAC: `users.read` for the user list and `redemptions.read` for redemption list/stat/detail. A session alone is insufficient and missing permission returns HTTP 403.
- Database-free Admin verification passed with a 768MB heap: TypeScript check, 2 suites / 8 tests and production build. PostgreSQL integration tests remain pending until `ADMIN_DATABASE_URL` points to an isolated test database.
