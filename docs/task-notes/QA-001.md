# QA-001 自动化验证基线

状态：部分完成（2026-07-12）

已建立 `npm run verify:demo` 冒烟验证，覆盖：

- welcome、登录、注册、KYC、首页、投资、组合、钱包、订单、通知、安全与关闭账户等关键路由。
- Polymarket 公共市场接口必须返回市场列表，并且 `tradingEnabled` 必须保持 `false`。
- 无效路由必须显示品牌化恢复页面和返回主页入口。
- Windows 构建固定使用单工作进程，避免本地环境的子进程创建耗尽导致不稳定构建。
