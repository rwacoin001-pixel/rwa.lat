# RWA.LAT 迭代与验收节奏

## 每次集成

1. 只修改当前任务范围，保留工作区内其他人的改动。
2. 更新对应 `docs/task-notes/<TASK-ID>.md`：已完成、未完成、外部依赖和验证证据。
3. 前端执行 `npm run build`；本地预览启动后执行 `npm run verify:demo`。
4. 涉及资金、身份或外部订单的功能必须注明 Demo/只读状态，不得用成功文案掩盖未接入的真实执行。

## 每周验收

- 产品：按 `docs/Task-Board.md` 的验收结果检查页面内容、交互和真实/演示边界。
- 设计：检查 390–430px 移动端、桌面手机壳、长文案、焦点、禁用、错误和空状态。
- 工程：检查构建、冒烟验证、环境变量隔离、敏感值未进入客户端或版本库。
- 风险：确认 PARTNER/LEGAL 外部阻塞未被误标为技术完成。

## 状态规则

- `待完成`：尚未开始。
- `进行中`：已经实现一部分，但没有形成可验收结果。
- `部分完成`：存在可运行结果，生产闭环仍有明确缺口。
- `待审核`：代码与验证证据齐备，等待产品负责人确认。
- `原型完成`：可运行的前端 Demo 已达到当前视觉和交互范围，不能等同生产完成。
- `外部阻塞`：依赖合同、法务、合作方或生产凭证。

## 发布门槛

真实 KYC、充值、提币、投资或 Polymarket 下单只在 API、账本、风控、审计、合作方和法务全部通过后开放；单独拥有 API 密钥不构成生产交易授权。
